Этот файл поможет быстро сориентироваться в материалах раздела `docs/ui-ux` и понять, где описаны маршруты, сценарии и вайрфреймы.

## Что где лежит

- Маршруты и сценарии (основной справочник): [routes-and-scenarios.md](./routes-and-scenarios.md)
- Вайрфреймы основных экранов: [wireframes.md](./wireframes.md)
- Общее описание и ссылки на Holst: [README.md](./README.md)

Примечание: каталог `mockups/` архивирован — используем Holst для визуализаций.

## Быстрые кейсы (где смотреть)

- Админ входит и создаёт студентов
  1. Прочитать «Подробные маршруты и действия (MVP) → Админ» в [routes-and-scenarios.md](./routes-and-scenarios.md).
  2. Ключевые экраны: `/auth/login`, `/admin/users` (штучное создание студента, импорт CSV), `/admin/groups` (группы).
- Преподаватель настраивает курс и принимает отправки
  1. Раздел «Преподаватель» в [routes-and-scenarios.md](./routes-and-scenarios.md).
  2. Ключевые экраны: `/t/courses`, `/t/courses/:courseId/labs`, `/t/integrations/github`, `/t/submissions/:submissionId`.
- Студент сдаёт лабу и смотрит результаты
  1. Раздел «Студент» в [routes-and-scenarios.md](./routes-and-scenarios.md) + вайрфреймы в [wireframes.md](./wireframes.md).
  2. Ключевые экраны: `/s/courses`, `/s/courses/:courseId/results`, `/s/labs/:labId`, `/s/submissions/:submissionId`.

## Навигация по MVP

- Аутентификация: локальная (email/пароль), JWT. OAuth через GitHub не используется для входа.
- GitHub интеграция: форк/PR — вручную на GitHub; приложение даёт ссылки и отслеживает PR вебхуками.
- Результаты студента: сводная таблица по курсу доступна по `/s/courses/:courseId/results`.
