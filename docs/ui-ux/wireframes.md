# Wireframes — Phase 1 (MVP Inline Adapter)

Низкоуровневые наброски экранов для студентов и преподавателей.

Публичная доска Holst (прототип):

## Студент — Лаба (Stepper «Сдать лабу»)

Сценарий: минимум шагов, максимум подсказок.

```
[Лаба: Вёрстка №1]
┌────────────────────────────────────────────────────────┐
│ Шаг 1: Вход                                           │
│  Email: [________ ]  Пароль: [________ ]  [ Войти ]   │
│  Нет аккаунта? [Зарегистрироваться]  Забыли пароль?   │
├────────────────────────────────────────────────────────┤
│ Шаг 2: Форк репозитория (GitHub)                      │
│  [Открыть репозиторий на GitHub]                      │
├────────────────────────────────────────────────────────┤
│ Шаг 3: Откройте страницу сравнения и создайте PR      │
│  [Открыть страницу PR (compare)]                      │
├────────────────────────────────────────────────────────┤
│ Структура проекта (пример)                            │
│  copy ⧉                                               │
│  .                                                     │
│  ├─ required_dir/                                      │
│  │   └─ index.html                                     │
│  └─ README.md                                          │
├────────────────────────────────────────────────────────┤
│ После PR статус появится автоматически:               │
│  • Получение файлов  • Проверка структуры  • Очередь  │
│  [Перейти к отправкам]                                 │
└────────────────────────────────────────────────────────┘
```

Пояснения:

- Крупные ссылки-переходы на GitHub: «Открыть репозиторий», «Открыть страницу PR (compare)».
- Привязка GitHub опциональна: достаточно указать username; OAuth не используется для входа.
- Пример структуры можно скопировать в буфер.
- Никаких терминов «clone», «checkout» в тексте — только при необходимости и в «Подробнее».
- Форк и Pull Request выполняются вручную на GitHub; приложение не создаёт их автоматически и лишь отслеживает события через вебхуки.

## Студент — Отправка (Статус + ошибки)

```
[Отправка #123  PR: https://github.com/org/repo/pull/45]
Статус: В очереди → Проверка структуры → Подготовка prompt → Вызов ИИ → Завершено

Базовые проверки:
✔ required_dir найдено
✖ отсутствует README.md — добавьте файл по образцу (ссылка)

[Задать вопрос преподавателю]   [Открыть PR]
```

## Пустые состояния и подсказки

- «Нет отправок» — покажите шаги: «Добавьте группу», «Отправьте уведомления студентам».
- «Нет прав доступа к репозиторию» — обновите токен GitHub в «Интеграциях».

## Доступность и мобильность

- Крупные кликабельные элементы, контраст, aria‑labels для кнопок «Форк», «PR».
- Тексты ошибок — чёткие инструкции «что сделать сейчас», без жаргона.

## Связь с фичами Phase 1

- Поддержка вебхука PR → автоматическое появление отправки в списке.
- Отображение прогресса inline-пайплайна (статусы шагов).
- Кнопка публикации предварительной оценки.
